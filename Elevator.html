Elevator Control Panel

System Prototype

Project Overview

This project is done in partial fulfillment of the requirements for the module CSD3515.

Design a control panel for users of an elevator for a building with 11 floors. The goal is to create an intuitive, efficient system that optimises elevator usage while ensuring a high-quality user experience.

Requirements

The control panel must support these interactions:

  • Open the doors
  • Close the doors
  • Select a destination floor from 1 to 11
  • Other relevant controls, displays, and feedback may be included

ILLUSTRATION

Control Panel

Wall Surface

The surface of the walls in the elevator are created using linear gradient tool with different shades of gray to create the metallic effect.

Buttons

Using the same color scheme as the walls, the buttons are created using angular gradient tool.

LED Screen

Digital number vectors are used to display the floor level on the LED screen.

Screws

Screws are created using the ellipse and polygon shape. Inner and drop shadows are used to add depth to the shapes, making it more realistic.

Environment

Lobby

When the door opens, users will see the lobby of the corresponding floor. Every floor has its own floor number sign and a pot of plant.

View Wireframes

FEATURES

Door Control

When the elevator is not moving, users can open and close the doors by pressing on the respective buttons.

Travelling

Elevator will bring the user to the corresponding level that was pressed. However, there can only be one target destination at each time for this prototype.

Floor Display

The black screen at the top of the control panel updates in real-time as the elevator moves. While the elevator is moving or will be moving to another level, the up or down arrow will light up accordingly.

View Final Prototype

CONDITIONS

  • Doors can open or close at any level when the elevator is not moving.
  • Whenever a button is pressed, it will light up.
  • If the door is open and the open button is pressed, the button will light up for a brief moment and return back to normal without doing anything. Same goes to when the door is closed and the close button is pressed.
  • When a floor button is pressed while the door is open, the door will close on its own automatically after 3s. Otherwise, you can also press the close button to immediately close the door and move onto the level.
  • When a floor button is pressed while the door is closed, the elevator will move to the next level immediately.
  • Buttons cannot be pressed while the elevator is on its way to another level.
  • The floor button cannot be pressed when the elevator is at that current level.
  • Functionality for the alarm button and any auditory feedback are currently not implemented for this prototype.
